# 学习路线

## 第一阶段：先把基础骨架搭起来

先理解下面几个主题，不追求一口气学完全部细节：

1. Claude API Messages
2. Tool Use
3. Message Batches
4. Agent SDK 基础
5. MCP 基础
6. Claude Code 基础

目标不是背文档，而是回答这 3 个问题：

- 这个东西是干什么的？
- 它在系统里处于哪一层？
- 什么时候该用它，什么时候不该用？

## 第二阶段：按考试领域重新整理

等前面的基础主题都过一遍后，再把知识映射回考试领域：

1. 代理架构与编排
2. 工具设计与 MCP 集成
3. Claude Code 配置与工作流
4. 提示工程与结构化输出
5. 上下文管理与可靠性

这样做比直接按领域硬啃更顺，因为领域本身是“考法”，不是“知识天然的组织方式”。

## 第三阶段：题目驱动复习

后面每做一道题，笔记里只补 2 类内容：

- 我原来没理解的概念
- 我容易混淆的边界

不要把题解全文搬进来，不然笔记会重新变成噪音。

## 当前顺序

我们先写：

1. `Claude API Messages 是什么`
2. `Tool Use 是什么`
3. `Message Batches 是什么`

这 3 个主题连得最紧，适合一起打底。
